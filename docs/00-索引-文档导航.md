1. [01-需求-需求文档.md](./01-需求-需求文档.md) 是原始需求与功能范围的最高依据，不因实现方便而增加或删除需求。
2. [07-设计-接口设计.md](./07-设计-接口设计.md) 与 [08-设计-数据库设计.md](./08-设计-数据库设计.md) 分别是目标 API 和数据模型事实来源。
3. 流程、架构、UI 与开发规则必须引用上述契约，不得自行发明接口、状态或数据表。
4. 当前真实完成度以代码为准：仅用户注册、登录、刷新令牌和退出登录完成；其他模块均为目标设计或参考脚手架。

## 范围分层

- **核心必做**：创作、Prompt、素材、草稿恢复、审核、质量评分、合规改写、站内发布、内容流、热点榜和爆文榜。
- **独立加分项**：点赞分享等用户反馈、外部热点导入、外部平台分发。加分项不进入核心主流程和基础 UI 验收。
- 不建设推荐榜、复杂分发中心、复杂运营后台、Redis、BullMQ、微服务或额外数据库。
