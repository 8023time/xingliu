# 前端架构

## 1. 应用职责

| 应用 | 技术栈 | 核心职责 |
| --- | --- | --- |
| `apps/creator` | Vite 8、React 19、React Router 7、Ant Design、Tiptap | 创作、草稿、审核、评分、改写和发布 |
| `apps/web` | Next.js 16、React 19、Tailwind CSS、Radix UI | 内容流、热点榜、爆文榜和公开详情 |

目标设计不等同于当前路由完成度。当前页面中的推荐、用户中心、后台榜单和 mock 数据不作为核心设计依据。

## 2. B 端目标页面

| 路由 | 职责 |
| --- | --- |
| `/login` | 登录与注册 |
| `/home` | 工作台：最近草稿、待处理内容、最近发布 |
| `/content/create`、`/content/:id/edit` | AI 候选、编辑器、草稿状态、审核与发布 |
| `/prompts` | Prompt CRUD |
| `/assets` | 素材上传、管理和基础合规状态 |
| `/contents` | 内容列表、状态、线上版本和编辑版本 |

审核结果、质量评分、合规改写和版本记录放在编辑器或内容详情面板，不拆独立审核页。核心阶段不建设复杂分发中心。

## 3. C 端目标页面

| 路由 | 职责 |
| --- | --- |
| `/` 或 `/explore` | 已发布内容流 |
| `/rankings` | 热点榜、爆文榜标签切换与无限滚动 |
| `/content/[id]` | 线上发布版本详情 |

核心阶段不建设推荐榜、搜索、用户中心、评论、收藏或相关推荐。

## 4. 数据与状态

- React Query 或 Server Component 管理服务端数据；Zustand 只管理客户端交互状态。
- Creator 使用 Dexie 保存本地草稿、同步队列和冲突记录。
- AI 生成结果先进入候选区，用户确认后写入 Tiptap。
- 内容列表和编辑器同时显示：
  - 线上状态：未发布、已发布或已下线。
  - 编辑版本状态：草稿、审核中、需改写、已拒绝或已通过。
- C 端接口只消费 `publishedVersion`，不得展示编辑中的 `currentVersion`。

## 5. 草稿同步

1. 用户停止输入约 2 秒后保存 IndexedDB。
2. 每 30 秒或关键操作后保存云端草稿快照。
3. 恢复网络后同步待处理快照。
4. 冲突时展示本地和云端内容，由用户决策，不自动合并。

## 6. 性能与验收

- C 端内容流和榜单使用游标分页与无限滚动，LCP 目标不高于 2.5 秒。
- 首屏不加载编辑器、图表或非当前页面重型模块。
- 所有页面覆盖加载、空、错误、禁用和无更多数据状态。
- 审核失败、评分缺失或审核服务不可用时，发布按钮禁用并说明原因。

## 7. 加分项边界

用户反馈、外部热点和外部分发只在核心闭环完成后增加简单入口，不进入核心路由、导航和基础验收。
