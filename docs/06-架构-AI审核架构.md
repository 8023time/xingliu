# AI 生成与审核架构

## 1. 技术职责

| 组件 | 职责 |
| --- | --- |
| 火山方舟模型 API | 提供 OpenAI 兼容的生成模型接口 |
| `@langchain/openai` `ChatOpenAI` | 调用火山方舟模型 API，完成图文生成、质量评分、合规改写 |
| `@langchain/core` | Prompt、消息、Runnable 和结构化输出基础能力 |
| Zod | 校验模型结构化输出 |
| `fastscan` | 本地敏感词快速检测 |
| `@alicloud/pop-core` | 调用阿里云内容安全增强版 |

AI 业务同步执行，不使用 Redis 或任务队列。火山方舟模型客户端和阿里云客户端均由内部 Service 单例复用。

`fastscan` 使用仓库内维护的基础敏感词文件 `server/libs/common/src/moderation/sensitive-words.ts`。基础词典用于本地高风险快速拦截，后续可通过代码评审持续扩展；不得把用户输入动态加入全局词典。

## 2. 四条链路

### 生成

```text
审核用户 Prompt
→ ChatOpenAI 生成结构化候选
→ Zod 校验
→ 审核生成结果
→ 返回候选区
```

候选结果不自动写入正式版本。生成前或生成后审核失败时，不返回可采纳候选。

### 安全审核

```text
Prompt / 内容版本 / 素材
→ fastscan
→ 阿里云内容安全增强版
→ 标准化 PASS / NEED_REWRITE / REJECT
→ 保存审核结果
```

本地高风险命中或阿里云高风险结果直接 `REJECT`。阿里云超时、鉴权失败、不可用或响应无法解析时，不产生 `PASS`。

### 质量评分

安全审核通过后，使用 `ChatOpenAI` 输出固定结构的总分、等级、多维评分、总结和改进建议。Zod 校验失败时记录任务失败，不保存正式评分，不允许发布。

### 合规改写

仅 `NEED_REWRITE` 内容可请求。`ChatOpenAI` 结合风险标签、命中片段和原因生成候选改写；用户采纳后创建新版本并重新执行完整审核与评分。

## 3. 标准化审核结果

```ts
interface ModerationResult {
  decision: 'pass' | 'need_rewrite' | 'reject';
  riskLevel: 'none' | 'low' | 'medium' | 'high';
  labels: string[];
  reason?: string;
  riskSpans?: unknown[];
  providerRequestId?: string;
  rawProviderOutput?: unknown;
}
```

- `high` 或本地高风险词：`reject`
- `medium`：`need_rewrite`
- `low` 或无风险：`pass`
- 服务异常：返回服务不可用错误，不映射为审核决策

## 4. 调用时机

| 时机 | 行为 |
| --- | --- |
| AI 生成前 | 审核用户 Prompt |
| AI 生成后 | 审核候选结果 |
| 素材上传后 | 更新素材最新合规状态 |
| 提交审核 | 审核新建的当前正式版本 |
| 审核通过后 | 执行质量评分 |
| 发布前 | 重新审核当前版本 |
| 二次编辑 | 新版本重新审核，不复用线上旧版本结果 |

## 5. 数据留痕

- `ai_tasks`：记录生成、审核、评分、改写任务状态、模型、耗时和脱敏摘要。
- `safety_reviews`：记录具体内容版本的标准化审核结果和阿里云原始结果。
- `quality_evaluations`：记录具体内容版本的正式评分。
- `rewrite_records`：记录候选改写和采纳状态。
- `assets`：只保存素材最新基础合规结果。

日志和 `ai_tasks` 不保存完整敏感正文、密码、令牌或密钥。

## 6. 环境变量

```env
ALIBABA_CLOUD_ACCESS_KEY_ID=
ALIBABA_CLOUD_ACCESS_KEY_SECRET=
ALIYUN_GREEN_ENDPOINT=https://green-cip.cn-shanghai.aliyuncs.com
ALIYUN_GREEN_TEXT_SERVICE=ai_art_detection
ALIYUN_GREEN_IMAGE_SERVICE=baselineCheck

OPENAI_API_KEY=
OPENAI_BASE_URL=
OPENAI_MODEL=
```

以上 `OPENAI_*` 变量是 `ChatOpenAI` 的兼容配置，实际填写火山方舟 API Key、OpenAI 兼容接口地址以及方舟接入点或模型标识。业务代码统一通过 `@langchain/openai` 调用火山方舟，不得直接调用原生 OpenAI SDK、自行发送模型 HTTP 请求或误接 OpenAI 官方服务。
