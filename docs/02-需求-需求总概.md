# 需求：项目概览与范围

## 项目定位

构建一个集 AIGC 辅助生产、AI 赋能的质量与安全管控、数据驱动的分发与优化于一体的创作者服务平台。

```text
创作者生产内容 -> AI 辅助生成 -> 草稿/版本保存 -> AI 审核 -> AI 质量评分 -> 发布 -> C端展示 -> 榜单排序 -> 外部分发
```

## 用户定位

- 内容消费用户：通过 C 端内容前台浏览首页信息流、热点榜、爆文榜、推荐榜和内容详情页。
- 内容创作者：通过 B 端创作者后台完成注册登录、创作工作台、Prompt 管理、素材管理、内容管理、审核结果查看和分发中心操作。

## 核心挑战

1. 构建 AIGC 内容创作的全链路体验：从灵感启动、Prompt 与素材管理、AI 图文生成、编辑、草稿保存、网络恢复，到最终发布。
2. 打造 AI 驱动的内容安全与质量防火墙：识别涉黄、涉赌、涉毒、敏感信息、低俗内容等风险，支持高危拦截、一键合规替代内容和质量评分。
3. 实现兼顾性能与智能排序的内容分发场景：提供热点榜、爆文榜、推荐榜，按内容质量、阅读热度、发布时间等维度排序，并保证榜单页 LCP 不高于 2.5 秒。

## 必做范围

1. 用户中心：支持手机号或邮箱注册、登录、退出。
2. Prompt 模板管理。
3. 多媒体素材管理，并对上传素材做基础合规性校验。
4. AI 图文内容生成，至少覆盖标题、正文、摘要、标签、配图建议。
5. 创意编辑器，支持 AI 生成内容二次编辑。
6. 内容一键发布。
7. 已发布内容支持二次编辑和更新。
8. 编辑过程中每 30 秒自动云端保存草稿。
9. 再次进入编辑器时自动恢复上次草稿。
10. 断网状态下编辑器继续工作，网络恢复后自动同步修改。
11. AI 内容审核，覆盖涉黄、涉赌、涉毒、敏感信息、低俗内容等风险。
12. 高危问题自动拦截或驳回。
13. 一键生成合规替代内容。
14. AI 内容质量多维度评分，并作为后续榜单排序依据。
15. 内容详情页，展示发布者、发布时间和具体图文内容。
16. 热点榜、爆文榜、推荐榜。
17. 榜单无限滚动加载。
18. 榜单页 LCP 不高于 2.5 秒。

## 进阶挑战

- 短图文创意编辑器：用户仅提供核心素材和大致思路，AI 自动生成结构完整、表达丰富的图文内容。
- 高精度内容安全识别：高危问题识别准确率达到 90% 以上，并支持内容下线、撤回、回滚。
- 智能排序的热点榜单：综合内容质量分、实时阅读热度、发布时间、用户反馈等动态加权因子。

## 成功标准

功能成功标准：

- 创作者可以从 B 端注册登录开始，完成一次 AI 创作、保存、审核、改写、评分、发布，并在 C 端榜单中看到内容。
- 离线编辑不会丢稿，恢复网络后可以同步或解决冲突。
- 高风险内容不能绕过审核直接发布。
- 榜单排序不是静态列表，至少能被内容质量分、阅读热度、发布时间影响；用户反馈作为进阶排序因子。

工程成功标准：

- 前后端职责清晰。
- 核心状态、DTO、AI 输出结构有共享类型或清晰 schema。
- API 有稳定错误码、requestId 和统一响应结构。
- 有基础测试策略、评估样例、部署说明和演示脚本。

## 交付清单

- 可运行系统：线上部署的创作者辅助生产与分发平台，提供可公开访问的 URL 或二维码。
- 技术文档：完整技术方案与架构设计文档，说明系统架构、关键技术选型、核心模块设计。
- 评估与总结：平台效果评估与内容分发优化报告，分析内容生成、审核效果和 Prompt 多次调教过程。
- 规则与体系：内容安全审核规则库与质量评估体系说明。
- 代码产出：托管于 GitHub 的完整项目代码，最终提交到 main 或 master 分支。
