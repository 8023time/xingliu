# 前端：应用架构与实现设计

## 1. 前端目标

前端分为两个应用：

- `apps/web`：C 端内容前台，面向内容消费用户，负责首页信息流、热点榜、爆文榜、推荐榜、内容详情和阅读互动。
- `apps/admin`：B 端创作者后台，面向内容创作者，负责登录注册、创作工作台、Prompt 管理、素材管理、内容管理、审核结果和分发中心。

B 端承载创作者生产主链路，C 端承载发布后的内容消费与榜单分发。

### 3.2 内容前台页面职责

- `/` 或 `/feed`：首页信息流，展示推荐内容、热点入口和最新发布内容。
- `/rankings/hot`：热点榜，强调阅读热度、发布时间衰减和质量分。
- `/rankings/viral`：爆文榜，强调阅读、点赞、分享等互动增长。
- `/rankings/recommend`：推荐榜，强调质量分、基础热度和用户反馈预留因子。
- `/content/[id]`：内容详情。

### 3.3 状态管理

C 端内容前台以服务端数据和轻量交互为主：

- `rankingStore`：当前榜单类型、滚动位置和筛选条件。
- `interactionStore`：点赞、分享、浏览去重等轻量状态。
- TanStack Query 管理首页信息流、榜单分页和内容详情缓存。

### 3.4 C 端性能

- 首页信息流和榜单首屏使用 SSR 或服务端预取。
- 首屏不加载编辑器和 AI 面板。
- 内容卡片图片固定比例。
- 使用 cursor 分页和无限滚动。
- LCP 目标不高于 2.5 秒。

### 4.2 创作者后台页面职责

- `/login`：创作者登录 & 注册。
- `/dashboard`：创作工作台，展示草稿、审核状态、发布内容和质量概览。
- `/create`：新建 AI 图文创作。
- `/create/:draftId`：编辑已有草稿，恢复本地或云端内容。
- `/drafts`：草稿箱，展示本地、云端、待同步、冲突状态。
- `/prompts`：Prompt 管理，支持系统 Prompt 和个人 Prompt。
- `/materials`：素材上传、素材库、标签和基础合规状态。
- `/contents`：内容管理，展示草稿、审核中、需改写、已发布、已更新内容。
- `/review/:contentId`：审核结果、质量评分、风险片段和合规改写。
- `/distribution`：分发中心，查看内容在热点榜、爆文榜、推荐榜中的表现和排序解释。

### 4.3 状态管理

B 端创作者后台建议使用 Zustand 管理本地交互状态：

- `authStore`：token、用户信息、登录状态。
- `editorStore`：当前草稿、标题、正文、素材、AI 候选内容。
- `syncStore`：网络状态、同步状态、冲突信息。
- `uiStore`：编辑器面板、弹窗、审核抽屉、分发中心筛选。

服务端数据使用 TanStack Query：

- 登录态和创作者信息。
- Prompt 列表。
- 素材列表。
- 草稿列表。
- 审核结果。
- 内容管理列表。
- 分发中心数据。

### 4.4 编辑器设计

- 富文本编辑器建议使用 Tiptap。
- AI 生成结果先进入候选区，不直接覆盖正文。
- 用户确认后支持插入、替换、追加。
- 编辑过程中持续触发本地保存和云端自动保存。
- 发布前必须经过审核结果校验。

### 4.5 离线能力

离线能力主要服务于 B 端创作者后台，详见 [07-前端-离线草稿与同步设计.md](./07-前端-离线草稿与同步设计.md)。前端必须保证：

- 断网时可以继续编辑。
- IndexedDB 中保留最新本地内容。
- 恢复网络后自动同步。
- 冲突时展示本地、云端、复制新草稿三种处理方式。

## 5. 前端验收

- C 端首页信息流、热点榜、爆文榜、推荐榜和内容详情页完整可用。
- B 端登录注册完整可用。
- B 端 AI 创作页可完成生成、编辑、保存、审核、发布。
- 断网编辑不丢稿。
- 审核结果和质量评分展示清晰。
- 榜单无限滚动流畅。
- 移动端主要页面无文字溢出和布局重叠。
