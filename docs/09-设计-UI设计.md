# UI 设计

## 1. 设计范围

UI 只服务三条核心链路：

1. 创作者使用 Prompt、素材和 AI 候选完成图文创作。
2. 创作者处理草稿、审核、质量评分、合规改写和站内发布。
3. 读者浏览已发布内容、热点榜和爆文榜。

不建设推荐榜、搜索、评论、私信、会员、复杂运营后台、复杂数据看板或复杂分发中心。

## 2. B 端页面

主导航只保留：

```text
工作台
创作
Prompt
素材
内容
```

| 页面 | 核心内容 |
| --- | --- |
| 登录注册 | 手机号或邮箱登录、注册、退出 |
| 工作台 | 开始创作、最近草稿、待处理内容、最近发布 |
| AI 创作编辑器 | 创作参数、AI 候选、正文编辑、草稿状态、审核与发布 |
| Prompt 管理 | 创建、编辑、删除和选择 Prompt |
| 素材管理 | 上传、查看、编辑、删除和合规状态 |
| 内容管理 | 内容状态、线上版本、编辑版本和当前操作 |

审核结果、评分、改写和版本记录放在编辑器或内容详情面板，不拆独立页面。

## 3. AI 创作编辑器

桌面端使用主编辑区与辅助面板：

- 主编辑区：标题、摘要、正文、封面。
- 辅助面板：创作设置、Prompt、素材、AI 候选、审核与评分。

创作设置只包含主题、内容类型、Prompt、素材以及可选受众、风格和关键词。内容类型仅为长文、短图文和短内容。

AI 生成结果必须先进入候选区，用户确认后才能插入、追加或替换正文。不得自动覆盖正文或自动发布。

编辑器固定展示：

- 返回内容管理。
- 当前内容标题。
- 草稿保存状态。
- 线上状态。
- 编辑版本状态。
- 提交审核与发布按钮。

## 4. 双版本状态展示

内容可以同时存在两个状态维度：

```text
线上状态：未发布 / 已发布 / 已下线
编辑版本：草稿 / 审核中 / 需改写 / 已拒绝 / 已通过
```

已发布内容二次编辑期间显示“旧版本仍在线”。新版本发布成功后提示线上版本已更新。下线后线上状态变为“已下线”，但编辑版本和历史版本仍可查看。

## 5. 草稿与审核交互

始终可见的草稿状态：

```text
本地已保存
云端同步中
云端已保存
离线编辑中
同步失败，点击重试
存在冲突
```

冲突时展示本地与云端版本，由用户选择保留内容。

审核面板展示结果、风险等级、标签、命中片段、原因、建议、质量总分和维度评分：

- 通过：允许发布。
- 需改写：突出合规改写入口。
- 拒绝：禁用发布，要求修改后重审。
- 审核服务不可用或评分缺失：禁用发布并提供重试。

合规改写先展示前后对比，用户采纳后创建新版本并重新审核。

## 6. C 端页面

主导航只保留内容流和榜单。榜单页使用“热点榜”和“爆文榜”标签切换。

- 内容卡片：封面、标题、摘要、作者、发布时间和阅读热度。
- 榜单卡片：可附加质量分或简短排序原因，不展示复杂算法细节。
- 内容详情：只展示线上发布版本的标题、作者、发布时间、封面和正文。
- 不增加推荐榜、用户中心、点赞、收藏、评论或相关推荐。

内容流和榜单必须覆盖首屏骨架、加载更多、失败重试、无更多内容和空状态。

## 7. 视觉与性能

- B 端使用现有 Ant Design 与 Tailwind，桌面端优先，关键操作和状态清晰。
- C 端内容优先、移动端友好，图片使用固定比例避免布局抖动。
- 状态不能只依赖颜色，必须同时显示文字。
- C 端 LCP 目标不高于 2.5 秒，首屏不加载编辑器或非必要重型模块。

## 8. 独立加分项 UI

核心闭环完成后，才可增加用户反馈、外部热点和外部分发的简单入口。加分项不得改变核心导航、主流程和基础验收。

## 9. 验收

- 创作者可以生成候选、确认写入、编辑、自动保存、恢复、审核、改写、评分和发布。
- 已发布内容可二次编辑，审核期间旧线上版本继续公开。
- 审核未通过、评分缺失或审核服务异常时无法发布。
- C 端只展示线上版本，可浏览内容流、热点榜、爆文榜和图文详情。
