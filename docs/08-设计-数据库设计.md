```text
用户
→ Prompt / 素材
→ 内容创作
→ 草稿自动保存
→ 内容版本
→ AI 审核
→ AI 质量评分
→ 合规改写
→ 发布
→ 榜单分发
→ 数据反馈
```

# 数据库：数据模型设计

## 一、用户与权限模块

### 1. users

用户基础账号表。

| 字段          | 类型/含义                   | 说明         |
| ------------- | --------------------------- | ------------ |
| id            | 主键                        | 用户 ID      |
| email         | 邮箱                        | 可为空，唯一 |
| phone         | 手机号                      | 可为空，唯一 |
| password_hash | 密码哈希                    | 登录凭证     |
| username      | 昵称                        | 展示名称     |
| avatar_url    | 头像                        | 可选         |
| status        | active / disabled / deleted | 用户状态     |
| created_at    | 创建时间                    | 自动填充     |
| updated_at    | 更新时间                    | 自动维护     |

索引：

- unique: `email`
- unique: `phone`
- index: `status`

> 建议不要强制 email 和 phone 都必填，用户可能只用其中一种注册。

---

### 2. user_roles

用户角色表。

| 字段       | 类型/含义                           | 说明        |
| ---------- | ----------------------------------- | ----------- |
| id         | 主键                                | 角色记录 ID |
| user_id    | 用户 ID                             | 外键        |
| role       | reader / creator / operator / admin | 角色        |
| created_at | 创建时间                            | 自动填充    |

索引：

- unique: `user_id + role`
- index: `role`

> 不建议在 `users` 里只放一个 `role`，因为一个用户可能既是创作者，也是读者。

---

# 二、Prompt 与素材模块

## 3. prompt_templates

Prompt 模板主表。

| 字段               | 类型/含义                   | 说明                                      |
| ------------------ | --------------------------- | ----------------------------------------- |
| id                 | 主键                        | Prompt 模板 ID                            |
| owner_id           | 创建人 ID                   | 系统模板可为空                            |
| name               | 模板名称                    | 如“小红书种草文案”                        |
| category           | 分类                        | article / image_text / marketing / review |
| description        | 模板说明                    | 可选                                      |
| visibility         | private / public / system   | 可见范围                                  |
| current_version_id | 当前版本 ID                 | 指向最新版本                              |
| usage_count        | 使用次数                    | 统计使用量                                |
| status             | active / disabled / deleted | 状态                                      |
| created_at         | 创建时间                    | 自动填充                                  |
| updated_at         | 更新时间                    | 自动维护                                  |

索引：

- index: `owner_id + updated_at`
- index: `category + visibility`
- index: `status`

---

## 4. prompt_template_versions

Prompt 模板版本表。

| 字段             | 类型/含义   | 说明                       |
| ---------------- | ----------- | -------------------------- |
| id               | 主键        | 版本 ID                    |
| prompt_id        | Prompt ID   | 外键                       |
| version_no       | 版本号      | 1、2、3                    |
| template         | Prompt 内容 | 实际模板文本               |
| variables_schema | 变量定义    | JSON，例如标题、风格、字数 |
| model_config     | 模型参数    | JSON，例如 temperature     |
| change_summary   | 变更说明    | 可选                       |
| created_by       | 创建人      | 外键                       |
| created_at       | 创建时间    | 自动填充                   |

索引：

- unique: `prompt_id + version_no`
- index: `created_at`

> 这样 AI 任务可以准确追溯当时使用的是哪个 Prompt 版本。

---

## 5. assets

素材表。

| 字段           | 类型/含义                               | 说明               |
| -------------- | --------------------------------------- | ------------------ |

| name           | 素材名称                                | 展示用             |

| mime_type      | 文件类型                                | image/png 等       |

| metadata       | 元数据                                  | JSON               |
| safety_status  | pending / pass / reject                 | 素材合规状态       |
| created_at     | 创建时间                                | 自动填充           |
| updated_at     | 更新时间                                | 自动维护           |
| deleted_at     | 删除时间                                | 软删除             |

索引：

- index: `user_id + created_at`
- index: `type`
- index: `safety_status`

| ------------- | ----------------------------- | ----------- |

| raw_ai_output | AI 原始输出                   | 可选        |

# 三、内容创作与版本模块

## 7. contents

内容主表。

| 字段               | 类型/含义                                                                    | 说明                    |
| ------------------ | ---------------------------------------------------------------------------- | ----------------------- |
| id                 | 主键                                                                         | 内容 ID                 |
| author_id          | 作者 ID                                                                      | 外键                    |
| content_type       | article / image_text / short_post                                            | 内容类型                |
| title              | 当前标题                                                                     | 冗余当前版本，方便查询  |
| summary            | 摘要                                                                         | 可选                    |
| cover_asset_id     | 封面素材 ID                                                                  | 可选                    |
| current_version_id | 当前版本 ID                                                                  | 指向 `content_versions` |
| status             | draft / reviewing / need_rewrite / rejected / approved / published / offline | 内容状态                |
| safety_status      | pending / pass / reject                                                      | 安全审核状态            |
| quality_level      | S / A / B / C / D                                                            | 质量等级                |
| quality_score      | 质量总分                                                                     | 用于排序                |
| safety_score       | 安全分                                                                       | 用于审核                |
| hot_score          | 热度分                                                                       | 用于热点榜              |
| recommend_score    | 推荐分                                                                       | 用于推荐榜              |
| published_at       | 发布时间                                                                     | 可选                    |
| offline_at         | 下线时间                                                                     | 可选                    |
| created_at         | 创建时间                                                                     | 自动填充                |
| updated_at         | 更新时间                                                                     | 自动维护                |
| deleted_at         | 删除时间                                                                     | 软删除                  |

索引：

- index: `author_id + updated_at`
- index: `status + published_at`
- index: `status + hot_score`
- index: `status + recommend_score`
- index: `quality_score`
- index: `published_at`

> `contents` 只存“当前状态”，完整历史放到 `content_versions`。

---

## 8. content_versions

内容版本表。

| 字段           | 类型/含义                          | 说明                     |
| -------------- | ---------------------------------- | ------------------------ |
| id             | 主键                               | 版本 ID                  |
| content_id     | 内容 ID                            | 外键                     |
| version_no     | 版本号                             | 1、2、3                  |
| title          | 标题                               | 当前版本标题             |
| summary        | 摘要                               | 可选                     |
| body           | 正文内容                           | Markdown / HTML / 富文本 |
| body_json      | 结构化内容                         | JSON                     |
| asset_ids      | 使用素材                           | JSON 数组                |
| cover_asset_id | 封面素材                           | 可选                     |
| change_type    | create / edit / rewrite / rollback | 变更类型                 |
| change_summary | 变更说明                           | 可选                     |
| created_by     | 创建人 ID                          | 外键                     |
| created_at     | 创建时间                           | 自动填充                 |

索引：

- unique: `content_id + version_no`
- index: `content_id + created_at`

> 发布后二次编辑、复审、回滚都依赖这张表。

## 9. draft_snapshots

草稿自动保存表。

| 字段            | 类型/含义                    | 说明         |
| --------------- | ---------------------------- | ------------ |
| id              | 主键                         | 草稿快照 ID  |
| content_id      | 内容 ID                      | 外键         |
| user_id         | 用户 ID                      | 外键         |
| base_version_id | 基于哪个内容版本编辑         | 可选         |
| title           | 草稿标题                     | 当前快照标题 |
| body            | 草稿正文                     | 当前快照内容 |
| body_json       | 结构化草稿                   | JSON         |
| asset_ids       | 素材列表                     | JSON         |
| client_revision | 客户端版本号                 | 用于离线同步 |
| server_revision | 服务端版本号                 | 用于冲突处理 |
| sync_status     | synced / pending / conflict  | 同步状态     |
| saved_from      | auto / manual / offline_sync | 保存来源     |
| created_at      | 创建时间                     | 自动填充     |

索引：

- index: `content_id + created_at`
- index: `user_id + created_at`
- index: `sync_status`

> 用它支撑“每 30 秒自动保存”和“断网恢复后同步”。

# 四、AI 审核、评分与改写模块

## 10. ai_tasks

AI 任务表。

| 字段              | 类型/含义                                                          | 说明                   |
| ----------------- | ------------------------------------------------------------------ | ---------------------- |
| id                | 主键                                                               | AI 任务 ID             |
| user_id           | 发起用户                                                           | 可为空，系统任务可为空 |
| content_id        | 内容 ID                                                            | 可选                   |
| asset_id          | 素材 ID                                                            | 可选                   |
| prompt_version_id | Prompt 版本 ID                                                     | 可选                   |
| task_type         | generate / moderate / score / rewrite / asset_check / image_prompt | 任务类型               |
| status            | pending / running / success / failed                               | 任务状态               |
| model_provider    | 模型供应商                                                         | 如火山方舟             |
| model_name        | 模型名称                                                           | 可选                   |
| input_summary     | 输入摘要                                                           | 避免直接存大文本       |
| output_summary    | 输出摘要                                                           | 可选                   |
| input_hash        | 输入哈希                                                           | 用于去重               |
| token_input       | 输入 Token                                                         | 可选                   |
| token_output      | 输出 Token                                                         | 可选                   |
| duration_ms       | 耗时                                                               | 可选                   |
| error_code        | 错误码                                                             | 可选                   |
| error_message     | 错误信息                                                           | 可选                   |
| created_at        | 创建时间                                                           | 自动填充               |
| updated_at        | 更新时间                                                           | 自动维护               |

索引：

- index: `task_type + status`
- index: `content_id + created_at`
- index: `user_id + created_at`

---

## 11. safety_reviews

内容安全审核表。

| 字段               | 类型/含义                    | 说明                     |
| ------------------ | ---------------------------- | ------------------------ |
| id                 | 主键                         | 审核记录 ID              |
| content_id         | 内容 ID                      | 外键                     |
| content_version_id | 被审核版本                   | 外键                     |
| ai_task_id         | AI 任务 ID                   | 外键                     |
| decision           | pass / need_rewrite / reject | 审核结论                 |
| risk_level         | none / low / medium / high   | 风险等级                 |
| risk_categories    | 风险分类                     | JSON，如涉黄、涉赌、涉毒 |
| risk_spans         | 命中片段                     | JSON                     |
| rule_hits          | 命中规则                     | JSON                     |
| safety_score       | 安全分                       | 0-100                    |
| ai_reason          | AI 判断理由                  | 可选                     |

| rule_set_version   | 规则集版本                   | 方便追溯                 |

## 13. quality_evaluation_dimensions

质量评分维度明细表。

| 字段           | 类型/含义 | 说明                              |

| id             | 主键      | 明细 ID                           |

# 核心状态流转

## 内容状态

```text
draft
→ reviewing
→ need_rewrite
→ reviewing
→ approved
→ published
→ offline
```

失败分支：

```text
reviewing → rejected
```

## AI 审核结果

```text
pass：通过
need_rewrite：需要改写
reject：拒绝发布
```

## 榜单排序核心因子

```text
推荐分 = 质量分 + 热度分 + 新鲜度分 + 互动分 - 风险惩罚分
```

# 最终建议

这套库表的核心设计思想是：

```text
contents 管当前状态
content_versions 管历史版本
draft_snapshots 管自动保存和离线同步
safety_reviews 管安全审核
quality_evaluations 管质量评分
ranking_scores 管榜单排序
ai_tasks 管 AI 调用审计
```
