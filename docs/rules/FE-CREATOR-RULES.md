# FE Creator AI Code Generation Rules

## 1. 产品边界

- `apps/creator` 是星流创作者中心，只实现工作台、AI 创作、Prompt、素材和内容管理。
- 审核、评分、改写和版本记录集成在编辑器或内容详情中。
- 不新增推荐榜、复杂分发中心、复杂数据看板、运营后台或需求之外的创作类型。
- 当前用户认证、Prompt、素材、内容壳、云端草稿、AI 候选生成与通用预检模块可视为已完成；不得用 mock 数据伪装其他接口已完成。具体完成范围与验证证据见 `docs/dev`。

## 2. 技术与结构

- 使用现有 React 19、React Router 7、Vite 8、TypeScript、Ant Design、Tailwind CSS、Tiptap、React Query、Zustand 和 Dexie。
- 服务端状态使用 React Query；客户端交互状态使用 Zustand；本地草稿和同步队列使用 Dexie。
- 页面通过 `api/` 或业务 hook 请求接口，不直接调用 axios。
- 接口路径和字段以 `docs/07-设计-接口设计.md` 为准；共享类型优先来自 `@xingliu/shared`。
- 不引入未安装依赖，不为单次使用过度封装组件、hook 或工具。

## 3. 核心交互

- AI 生成结果先进入候选区，用户确认后才能写入编辑器。
- 内容类型仅为长文、短图文和短内容。
- 停止输入约 2 秒后保存本地草稿，每 30 秒保存云端快照。
- 冲突时展示本地和云端内容，不静默覆盖。
- 同时展示线上状态和编辑版本状态；二次编辑时明确旧版本仍在线。
- 审核未通过、评分缺失或审核服务异常时禁用发布并说明原因。
- 合规改写先展示前后对比，采纳后重新审核。

## 4. UI 与代码

- 优先复用 Ant Design，不包装无业务含义的基础组件。
- 页面与组件完整处理加载、空、错误、禁用、离线和冲突状态。
- 状态不得只靠颜色表达；按钮和图标提供可访问名称。
- 不保存或输出明文密码、令牌、密钥和审核原始响应。
- 不使用 `any` 逃避类型，不手改 Orval 或其他生成代码。

## 5. 验证

代码修改后运行：

```bash
pnpm --filter @xingliu/creator build
```

仅修改文档时检查 UTF-8、接口路径、状态和核心流程一致性。
