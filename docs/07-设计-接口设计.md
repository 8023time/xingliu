# 后端接口设计

## 1. 通用规范

- 全局前缀：`/api`
- 保留已完成用户接口 `/api/user/*`；新增资源使用复数路径。
- 写接口默认需要 JWT；公开内容、内容流和榜单无需登录。
- 列表参数为 `cursor?: string`、`limit?: number`，默认 20、最大 50。

```ts
interface CursorPage<T> {
  items: T[];
  nextCursor: string | null;
  hasMore: boolean;
}
```

接口完成度以代码和 `docs/dev` 为准。当前用户、Prompt、素材、内容壳、云端草稿、AI 候选生成与通用预检接口已完成；其他接口仍为目标契约。

## 2. 已完成用户接口

| 方法 | 路径 | 鉴权 | 说明 |
| --- | --- | --- | --- |
| POST | `/api/user/register` | 否 | 注册 |
| POST | `/api/user/login` | 否 | 登录 |
| POST | `/api/user/refreshToken` | 否 | 刷新令牌 |
| POST | `/api/user/logout` | 是 | 退出登录 |

## 3. Prompt 与素材

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET、POST | `/api/prompts` | 查询、创建 Prompt |
| GET、PATCH、DELETE | `/api/prompts/:id` | 查询、更新、软删除 Prompt |
| GET、POST | `/api/assets` | 查询、上传或创建链接素材 |
| GET、PATCH、DELETE | `/api/assets/:id` | 查询、更新、软删除素材 |
| POST | `/api/assets/:id/moderation` | 重新审核素材 |

## 4. 内容生成

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/api/ai/generate` | 审核输入并生成经过审核的候选图文 |
| POST | `/api/moderation/check` | 通用 Prompt、文本或素材预检 |

AI 候选生成请求：

```ts
interface AiGenerateRequest {
  contentId: string;
  topic: string;
  promptId: string;
  assetIds?: string[];
  audience?: string;
  style?: string;
  keywords?: string[];
}
```

每次固定生成 3 个候选，响应候选结构为：

```ts
interface AiGenerateCandidate {
  id: string;
  title: string;
  summary: string;
  body: string;
  tags: string[];
}
```

候选只返回到候选区，不自动写入草稿。输入或任一候选审核失败时，不返回可采纳候选。

```ts
interface ModerationCheckRequest {
  subjectType: 'prompt' | 'content' | 'asset';
  subjectId?: string;
  text?: string;
  assetUrl?: string;
}
```

`text` 与 `assetUrl` 至少提供一个。审核服务异常返回错误，不返回 `pass`。

## 5. 内容、草稿与版本

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/api/contents` | 创建内容壳 |
| GET | `/api/contents` | 查询我的内容 |
| GET、PATCH、DELETE | `/api/contents/:id` | 查询汇总信息、更新元数据、软删除 |
| GET | `/api/contents/:id/versions` | 查询正式版本 |
| POST | `/api/contents/:id/drafts` | 保存云端草稿快照 |
| GET | `/api/contents/:id/drafts/latest` | 获取最新云端草稿 |
| POST | `/api/contents/:id/drafts/sync` | 同步离线草稿 |

创建内容壳：

```ts
{
  contentType: 'article' | 'image_text' | 'short_post';
  title?: string;
}
```

草稿保存正文、标题、摘要、素材引用和修订号。草稿不会自动成为正式版本。冲突时“复制为新草稿”通过先创建新的内容壳，再为新内容保存首个草稿快照实现。

## 6. 审核、评分、改写与发布

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/api/contents/:id/review` | 将指定草稿提升为当前正式版本并审核、评分 |
| POST | `/api/contents/:id/quality-evaluation` | 当前版本评分失败后的重试 |
| POST | `/api/contents/:id/compliance-rewrite` | 生成合规改写候选 |
| POST | `/api/contents/:id/compliance-rewrite/:rewriteId/accept` | 采纳改写并创建新版本 |
| POST | `/api/contents/:id/publish` | 发布前复审并切换线上版本 |
| POST | `/api/contents/:id/offline` | 清空线上版本并下线 |

提交审核：

```ts
{
  draftSnapshotId: string;
  changeSummary?: string;
}
```

发布前必须满足：当前版本审核通过、当前版本存在质量评分、发布前复审成功。发布成功后 `publishedVersionId = currentVersionId`。

## 7. 内容消费与榜单

| 方法 | 路径 | 鉴权 | 说明 |
| --- | --- | --- | --- |
| GET | `/api/feed` | 否 | 已发布内容流 |
| GET | `/api/public/contents/:id` | 否 | 线上版本详情 |
| GET | `/api/rankings/hot` | 否 | 站内热点榜 |
| GET | `/api/rankings/viral` | 否 | 爆文榜 |

公开响应必须返回 `publishedVersion` 内容，禁止返回尚未发布的 `currentVersion`。核心榜单仅使用质量分、阅读热度和发布时间。

## 8. 独立加分项接口

以下接口不进入核心主流程和基础验收：

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/api/contents/:id/interactions` | 阅读、点赞、分享等反馈 |
| GET | `/api/topics/hot` | 查询外部热点 |
| POST | `/api/topics/import` | 导入外部热点 |
| POST、GET | `/api/contents/:id/distributions` | 创建、查询外部分发任务 |

外部分发失败只更新任务状态，不回滚站内发布。
