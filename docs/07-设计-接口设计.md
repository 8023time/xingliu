# 后端：API 接口规范

## 一、通用规范

### 响应统一规范

| 响应类型 | code               | success | 说明                                     |
| -------- | ------------------ | ------- | ---------------------------------------- |
| 业务成功 | 0                  | true    | data 为成功数据，如无数据返回 null       |
| 业务失败 | 1                  | true    | data 为详细错误描述，可不返回或返回 null |
| 系统错误 | 401/403/404/500... | false   | 未授权/禁止访问/未找到/服务器错误等      |

**完整响应示例：**

业务成功：

```json
{
  "timestamp": "2026-05-22T10:00:00Z",
  "path": "/api/contents",
  "message": "请求成功",
  "code": 0,
  "data": {}, // 详细数据
  "success": true
}
```

业务失败：

```json
{
  "timestamp": "2026-05-22T10:00:00Z",
  "path": "/api/contents",
  "message": "错误描述",
  "code": 1,
  "data": {}, // 详细业务错误信息
  "success": true
}
```

系统错误：

```json
{
  "timestamp": "2026-05-22T10:00:00Z",
  "path": "/api/contents",
  "message": "错误描述",
  "code": 401,
  "success": false
}
```

通用约定：

- B 端部分生产侧写接口需要 JWT 鉴权；C 端阅读互动接口需要 JWT;
- 所有的用户都有 C 端与 B 端角色与权限，通过接口鉴权区分；后续可增加管理员角色。
- B 端创作者后台调用注册登录、Prompt、素材、草稿、AI 创作、审核、评分、改写、发布和分发中心接口。
- C 端内容前台调用内容详情、首页信息流、热点榜、爆文榜、推荐榜和阅读互动接口。
- 所有的接口都必须经过 DTO 校验，而且需要在 Swagger 中有完整的接口定义和示例。
- AI 生成、审核、评分、改写在 MVP 阶段优先同步返回；耗时增加时再切换为任务模式。

### 2. Prompt 模板模块接口

Prompt 模板用于 LangChain 调用前的变量注入、版本追踪和效果复盘。模板、变量 schema、模型参数写入 PostgreSQL；AI 任务只引用确定的 `promptVersionId`，避免历史任务被后续模板修改影响。

| 方法   | 路径                         | 说明                 | 鉴权 | 返回                   |
| ------ | ---------------------------- | -------------------- | ---- | ---------------------- |
| GET    | /api/prompts                 | 获取 Prompt 模板列表 | ✅   | Page\<PromptTemplate\> |
| GET    | /api/prompts/:id             | 获取 Prompt 模板详情 | ✅   | PromptTemplate         |
| POST   | /api/prompts                 | 新建 Prompt 模板     | ✅   | PromptTemplate         |
| PUT    | /api/prompts/:id             | 更新 Prompt 基础信息 | ✅   | PromptTemplate         |
| POST   | /api/prompts/:id/versions    | 创建 Prompt 新版本   | ✅   | PromptVersion          |
| GET    | /api/prompts/:id/versions    | 获取 Prompt 版本列表 | ✅   | PromptVersion[]        |
| POST   | /api/prompts/:id/set-current | 切换当前版本         | ✅   | PromptTemplate         |
| POST   | /api/prompts/:id/use         | 记录模板使用         | ✅   | PromptUsageResult      |
| DELETE | /api/prompts/:id             | 软删除 Prompt 模板   | ✅   | -                      |

#### 2.1 Prompt 列表接口 `GET /api/prompts`

**查询参数：**

| 字段       | 类型   | 必需 | 说明                                 |
| ---------- | ------ | ---- | ------------------------------------ |
| cursor     | string | 否   | 游标分页                             |
| limit      | number | 否   | 默认 20，最大 50                     |
| category   | string | 否   | article / image_text / review 等分类 |
| visibility | string | 否   | private / public / system            |
| keyword    | string | 否   | 按名称和描述搜索                     |

#### 2.2 创建 Prompt 版本接口 `POST /api/prompts/:id/versions`

**请求：**

| 字段            | 类型     | 必需 | 说明                                    |
| --------------- | -------- | ---- | --------------------------------------- |

| modelConfig     | object   | 否   | provider、modelName、temperature 等参数 |
| changeSummary   | string   | 否   | 版本变更说明                            |

**请求示例：**

```json

    { "name": "topic", "type": "string", "required": true },
    { "name": "audience", "type": "string", "required": true },
    { "name": "style", "type": "string", "required": false }

    "provider": "openai",
    "modelName": "gpt-4.1-mini",
    "temperature": 0.7
  },
  "changeSummary": "补充受众变量"
}
```

### 4. 草稿自动保存与同步接口

| 方法 | 路径                                     | 说明             | 鉴权 | 返回            |

| POST | /api/contents/:id/draft-snapshots/sync   | 离线草稿同步     | ✅   | DraftSyncResult |

#### 4.1 保存草稿快照接口 `POST /api/contents/:id/draft-snapshots`

| 方法   | 路径                            | 说明             | 鉴权 | 返回               |
| ------ | ------------------------------- | ---------------- | ---- | ------------------ |
| POST   | /api/contents                   | 新建草稿内容     | ✅   | Content            |
| GET    | /api/contents                   | 获取我的内容列表 | ✅   | Page\<Content\>    |
| GET    | /api/contents/:id               | 获取内容详情     | ✅   | ContentDetail      |
| PUT    | /api/contents/:id               | 更新内容当前版本 | ✅   | Content            |
| POST   | /api/contents/:id/versions      | 创建内容新版本   | ✅   | ContentVersion     |
| GET    | /api/contents/:id/versions      | 获取内容版本列表 | ✅   | ContentVersion[]   |
| POST   | /api/contents/:id/rollback      | 回滚到指定版本   | ✅   | Content            |
| POST   | /api/contents/:id/submit-review | 提交审核         | ✅   | ReviewSubmitResult |
| POST   | /api/contents/:id/publish       | 发布内容         | ✅   | PublishResult      |
| POST   | /api/contents/:id/offline       | 下线内容         | ✅   | Content            |
| DELETE | /api/contents/:id               | 软删除内容       | ✅   | -                  |

#### 6.1 新建草稿内容接口 `POST /api/contents`

**请求：**

| 字段          | 类型     | 必需 | 说明                                   |
| ------------- | -------- | ---- | -------------------------------------- |
| contentType   | string   | ✓    | article / image_text / short_post      |
| title         | string   | ✓    | 内容标题，对应 `contents.title`        |

- 创建 `contents` 记录，默认 `status = draft`、`safety_status = pending`。


#### 6.2 发布内容接口 `POST /api/contents/:id/publish`

**请求：**

| 字段             | 类型     | 必需 | 说明                       |
| ---------------- | -------- | ---- | -------------------------- |
| contentVersionId | string   | 否   | 指定发布版本               |
| publishAt        | string   | 否   | 定时发布时间，MVP 可不实现 |
| channels         | string[] | 否   | internal / toutiao 等渠道  |

**发布约束：**

- 必须登录。
- 当前用户必须是内容作者或具备运营权限。
- 内容状态必须是 `approved`。
- 最新审核记录 `decision` 必须是 `pass`。
- 最新质量评分必须存在，`level` 不建议低于 `B`。
- 发布成功后初始化或更新 `content_metrics` 和 `ranking_scores`。

### 7. 审核、评分与合规改写接口

审核、评分和改写都通过 AI Gateway 调用 LangChain + OpenAI，并把结构化结果写入 PostgreSQL。审核结果决定能不能发布，质量评分决定分发权重，合规改写只生成候选版本，不自动覆盖原文。

| 方法 | 路径                               | 说明                 | 鉴权 | 返回                   |
| ---- | ---------------------------------- | -------------------- | ---- | ---------------------- |
| POST | /api/moderation/check              | 内容安全审核         | ✅   | ModerationResult       |
| GET  | /api/moderation/records/:contentId | 获取内容审核记录     | ✅   | ModerationRecord[]     |
| GET  | /api/moderation/records/detail/:id | 获取审核记录详情     | ✅   | ModerationRecordDetail |
| POST | /api/quality/score                 | 内容质量评分         | ✅   | QualityEvaluation      |
| GET  | /api/quality/:contentId            | 获取内容评分记录     | ✅   | QualityEvaluation[]    |
| POST | /api/rewrite/compliance            | 生成合规改写候选     | ✅   | RewriteResult          |
| POST | /api/rewrite/:id/accept            | 采纳改写并生成新版本 | ✅   | ContentVersion         |

#### 7.1 内容安全审核接口 `POST /api/moderation/check`

**请求：**

| 字段             | 类型   | 必需 | 说明                                      |
| ---------------- | ------ | ---- | ----------------------------------------- |
| contentId        | string | ✓    | 内容 ID，对应 `safety_reviews.content_id` |
| contentVersionId | string | 否   | 内容版本 ID，不传则使用当前版本           |
| ruleSetVersion   | string | 否   | 审核规则集版本，不传则使用当前启用版本    |

```json
  "aiTaskId": "task_002",
  "decision": "need_rewrite",
  "riskLevel": "medium",
  "safetyScore": 68,
  "categories": [
    {
      "category": "spam_ad",
      "confidence": 0.88,
      "reason": "存在站外导流倾向"
    }
  ],
  "riskSpans": [
    {
      "text": "加我领取资料",
      "start": 120,
      "end": 126,
      "category": "spam_ad",

    }
  ],
```

**落库规则：**

- 创建 `safety_reviews`，写入 `decision`、`risk_level`、`risk_categories`、`risk_spans`、`rule_hits`、`safety_score` 和 `rule_set_version`。
- 更新 `contents.safety_status`、`contents.safety_score`，并根据审核结论将内容状态更新为 `approved`、`need_rewrite` 或 `rejected`。
- 审核调用产生的模型信息、耗时和 token 统一写入 `ai_tasks`，并通过 `ai_task_id` 关联。

#### 7.2 内容质量评分接口 `POST /api/quality/score`

**请求：**

| 字段             | 类型   | 必需 | 说明         |
| ---------------- | ------ | ---- | ------------ |
| contentId        | string | ✓    | 内容 ID      |
| contentVersionId | string | 否   | 内容版本 ID  |
| rubricVersion    | string | 否   | 评分标准版本 |

**落库规则：**

- 创建 `quality_evaluations` 主记录，写入 `total_score`、`level`、`rubric_version`、`summary`、`improvements`。
- 创建多条 `quality_evaluation_dimensions`，保存各维度分数、权重和评分理由。
- 更新 `contents.quality_score` 和 `contents.quality_level`，用于榜单排序。

#### 7.3 合规改写接口 `POST /api/rewrite/compliance`

**请求：**

| 字段            | 类型   | 必需 | 说明                                       |
| --------------- | ------ | ---- | ------------------------------------------ |
| contentId       | string | ✓    | 内容 ID，对应 `rewrite_records.content_id` |
| sourceVersionId | string | ✓    | 原始内容版本，对应 `source_version_id`     |
| reason          | string | 否   | 改写原因，对应 `reason`                    |

**落库规则：**

- 调用 AI Gateway 生成合规标题、正文和修改片段。
- 创建 `rewrite_records`，保存 `source_title`、`source_body`、`rewritten_title`、`rewritten_body`、`changed_spans`。
- 默认 `accepted = false`，不自动覆盖正文。
- 用户调用 `POST /api/rewrite/:id/accept` 后，创建新的 `content_versions`，`change_type = rewrite`，并回写 `rewrite_records.rewritten_version_id`。

### 12. 接口实现约定

接口与 Prisma 模型映射：

| 接口模块      | 主要落库模型                                                                                 |
| ------------- | -------------------------------------------------------------------------------------------- |
| 用户模块      | `User`, `UserRole`                                                                           |
| Prompt 模板   | `PromptTemplate`, `PromptTemplateVersion`                                                    |
| 素材模块      | `Asset`, `AssetSafetyCheck`                                                                  |
| 内容与草稿    | `Content`, `ContentVersion`, `DraftSnapshot`                                                 |
| AI 创作与审计 | `AiTask`                                                                                     |
| 审核模块      | `SafetyReview`, `AuditRuleSet`, `AuditRule`                                                  |
| 质量评分      | `QualityEvaluation`, `QualityEvaluationDimension`, `QualityRubric`, `QualityRubricDimension` |
| 合规改写      | `RewriteRecord`, `ContentVersion`                                                            |
| 榜单与互动    | `ContentMetric`, `ContentInteraction`, `RankingScore`, `HotTopic`, `ContentTopicRelation`    |
| 分发模块      | `DistributionChannel`, `DistributionTask`                                                    |
| 内容操作审计  | `ContentOperation`                                                                           |

- 后端使用 NestJS Module 拆分：Auth、User、Prompt、Asset、Content、DraftSnapshot、Moderation、Quality、Rewrite、Ranking、Distribution、AiGateway。
- LangChain + OpenAI 只允许在 `AiGatewayModule` 内部调用，业务模块通过服务方法调用 AI Gateway。
- 所有 AI 输出必须做结构化校验，校验失败最多请求模型修复一次。
- 所有 AI 调用必须写入 `ai_tasks`，并记录模型、Prompt 版本、输入摘要、输出摘要、token 和耗时。
- 所有列表接口统一使用游标分页，返回 `items`、`nextCursor`、`hasMore`。
- PostgreSQL 中 JSON 数据优先使用 `jsonb`，用于正文结构、变量 schema、风险片段、评分维度、素材元数据。
- 创建内容版本、审核记录、评分记录、发布状态变更必须使用事务，避免状态和记录不一致。
- 发布、点赞、收藏、分发任务创建等写接口必须支持幂等键 `Idempotency-Key`。
- C 端公开接口只能返回已发布、未删除、未下线内容。
- B 端写接口必须校验作者身份或角色权限，禁止越权访问其他创作者内容、草稿快照和素材。
- 接口不新增 `CreatorProject` 等 schema 外模型；创作工作台的“项目”概念由 `contents(status=draft)` 和 `draft_snapshots` 组合支撑。
