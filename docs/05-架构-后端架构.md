# 后端架构

## 1. 目标与当前状态

`server` 是统一 NestJS 模块化单体 API，共享 PostgreSQL 和 MinIO，不使用 Redis、BullMQ 或微服务。

当前用户认证、Prompt、素材、内容壳、云端草稿、AI 候选生成与通用预检模块已完成。其他已有模块仍为待实现脚手架；榜单代码仅作参考，目标路径和行为以接口设计为准。具体完成范围与验证证据见 `docs/dev`。

## 2. 模块职责

| 模块 | 核心职责 |
| --- | --- |
| `user`、`auth` | 已完成的用户与双令牌认证 |
| `prompt` | Prompt CRUD |
| `asset` | 素材上传、管理和基础合规检查 |
| `content` | 内容壳、正式版本、发布、下线 |
| `draft` | 云端快照、恢复、离线同步和冲突 |
| `ai-generation` | `ChatOpenAI` 图文候选生成 |
| `moderation` | `fastscan` 与阿里云审核编排 |
| `quality` | 质量评分 |
| `rewrite` | 合规改写候选与采纳 |
| `ranking` | 内容流、热点榜和爆文榜 |

`topic`、`distribution` 与互动能力属于独立加分项。

## 3. 版本与公开语义

- 创建内容时先创建内容壳，不要求立即创建正式版本。
- 草稿只写入 `draft_snapshots`。
- 提交审核时将指定草稿提升为不可变 `content_versions`，并设置 `currentVersionId`。
- `currentVersionId` 表示创作者当前处理版本。
- `publishedVersionId` 表示 C 端线上版本。
- 二次编辑时旧线上版本继续公开；新版本通过审核并发布后才替换线上版本。
- 下线时清空 `publishedVersionId`。
- 公开详情、内容流和榜单必须读取 `publishedVersion`。

## 4. 请求与响应

- 全局前缀为 `/api`。
- 保留已完成的 `/api/user/*`；新增资源使用复数路径。
- 写接口默认使用 JWT，并校验资源归属。
- DTO 使用 `class-validator`；模型结构化输出使用 Zod。
- 列表统一使用 `cursor`、`limit`、`nextCursor` 和 `hasMore`。
- 使用现有全局响应拦截器和异常过滤器，不把系统错误伪装成成功。

## 5. 事务边界

以下操作使用短 Prisma 事务：

- 提交草稿并创建正式版本。
- 保存审核结果并更新当前状态。
- 保存评分并更新当前汇总分数。
- 采纳改写并创建新版本。
- 发布并切换 `publishedVersionId`、初始化指标。
- 下线并清空 `publishedVersionId`。

外部 AI、阿里云和对象存储调用不得放入数据库事务。先完成外部调用，再用短事务保存结果。

## 6. AI 与审核

- 生成、评分、改写统一通过内部 AI Service 使用 `@langchain/openai` 的 `ChatOpenAI`，并通过火山方舟提供的 OpenAI 兼容接口调用模型。
- 内容安全统一通过内部 Moderation Service 使用 `fastscan` 和 `@alicloud/pop-core`。
- 业务模块不得直接创建模型或阿里云客户端。
- 高风险判断直接拒绝；模型结果不得覆盖阿里云高风险结果。
- 审核服务异常或评分缺失时禁止发布。

## 7. 榜单

核心热点榜和爆文榜根据以下数据实时计算，不保存排名快照：

```text
质量分 + 阅读热度 + 发布时间
```

用户反馈仅作为进阶智能排序因子。公开查询必须筛选存在 `publishedVersionId` 且未删除、未下线的内容。

## 8. 配置与验证

敏感配置只从环境变量读取。数据库、接口或日志不得保存凭证和完整敏感正文。

后端相关修改至少执行：

```bash
pnpm --filter @xingliu/server exec prisma validate
pnpm --filter @xingliu/server build
```
